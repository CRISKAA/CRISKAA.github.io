2.NOIONI

1.1 题目描述

NOIONI 是 NOI 的叔叔。NOIONI 先生很受欢迎，因为他的名字中 N,
O, I 各重复了两遍。最近 NOIONI 先生生了一个孩子（吓）。现在 NOIONI
先生打算给他的孩子取名字，他希望他的孩子的名字也由 N, O, I 组成，而
且每个字母重复的次数相同。NOIONI 先生找来了一个世代相传的手卷，
手卷里写着一首由 N, O, I 组成的 N 个字符的诗，NOIONI 先生打算从这
个手卷里找出一个最长的子串，使得这个子串符合每个字母出现次数相同，
并且尽可能的长，以此作为儿子的名字。

1.2 输入格式

第一行输入一个整数 N 表示诗的长度。
第二行输入一个由 N, O, I 组成的字符串 S, 表示这首诗。

1.3 输出格式

输出一个数字，表示最长的每个字符出现次数相同的子串的长度。如
果没有，输出 0.

1.4 样例解释

NOIINONOOI 中最长的每个字符出现次数相同的子串是 IINONO，所
以输出 6.
IOIINIIO 中没有符合要求的子串，输出 0.

1.5 数据规模及约定

对于 10% 的数据，有 N <= 200,
对于另外 15% 的数据，有 N <= 4 000,
对于所有的数据有 N <=200 000.

用N[X] ,O[X], I[X]表示前X个数中N,O,I的个数，那么假设从X到Y的字符串条件符合，可以想到的是，前Y个字符的N,O,I之差必定和前X个字符的N,O,I之差相等，并且我们是从前往后搜的，前X个字符的值已经计算过了，所以我们可以用map来记录位置，每次判断一下N,O,I之差相等的字符串的位置（没有则是0），这样时间复杂度就能减少到n*log（n）了，因为map一次计算为log(n)。
